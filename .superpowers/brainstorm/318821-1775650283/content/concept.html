What kind of app should this be?

The source is a rich field guide ("cuaderno de campo") for young naturalists — 7 missions, species ID, geology, ethnobotany, orienteering. How faithful to that concept should the app be?

🗺️ trail.html + Boniches content
Same map UI as Hanover/Allue routes
• Map + GPS tracking
• POI cards with species info
• Image carousel at each stop
• Same look as existing routes

A — Reuse trail.html

Plug Boniches data into the existing map viewer. Fast, consistent with other routes. Loses the "field guide" feel but gains GPS, offline tiles, elevation profile.

📓 Cuaderno de campo digital
New page styled like a field notebook
• Mission-by-mission narrative
• Species cards with illustrations
• Mini map per mission (not full GPS)
• "Gamified" mission completion

B — Digital field guide

A new bespoke page that captures the educational, adventure feel of the cuaderno. Scrollable missions, species ID panels, rich illustrations. Less GPS, more storytelling.

🧭 Map + Field Guide hybrid
Full map with rich POI panels
• Full Leaflet map + GPS
• POI modal = mini field-guide page
• Species grids, geology panels
• Mission-themed stop names

C — Hybrid: map + rich modals

Use the existing map infrastructure but make POI modals much richer — tabbed sections for species, geology, ethnobotany. Best of both worlds but more complex to build.
